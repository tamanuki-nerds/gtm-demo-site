# レクチャー①：GA4とGTMの基礎知識

> 所要時間：約30分（読み物）

---

## 1. GA4とは？

**Google Analytics 4（GA4）** は、Googleが提供する無料のアクセス解析ツールです。
サイトやアプリに来たユーザーが「何をしたか」を計測・レポートします。

広告代理店で使う主な場面：
- 広告からの流入が購入やCV（コンバージョン）につながっているか確認する
- どのページで離脱が多いか把握してランディングページ改善に活かす
- 広告媒体（Google広告、Meta広告など）と連携してROASを計算する

### GA4の3つの基本概念

| 概念 | 説明 | 例 |
|------|------|-----|
| **プロパティ** | 計測対象のサイト・アプリ1つを管理する単位 | 「STYLE DEMOのGA4プロパティ」 |
| **データストリーム** | 計測データの入口。ウェブ/iOS/Androidごとに作る | 「https://example.com のウェブストリーム」 |
| **イベント** | ユーザーの行動1回1回の記録 | ページを見た（page_view）、ボタンを押した（click）など |

### GA4のイベントの種類

```
自動収集イベント      → GA4が最初から自動で計測する（page_view, scroll, clickなど）
拡張計測イベント      → GA4管理画面でONにするだけで計測できる（外部リンク離脱など）
                        ※ 設定場所：GA4 → 管理 → データストリーム → 拡張計測
推奨イベント          → Googleが名前を定義している（purchase, add_to_cartなど）
カスタムイベント      → 自分で名前をつけて計測する（GTMからdataLayerで送る）
```

## 2. GTMとは？

**Google Tag Manager（GTM）** は、タグ（計測コード）をサイトに埋め込むための管理ツールです。

### なぜGTMを使うの？

GTMがない時代：
1. GA4を入れたい → エンジニアに依頼 → 数日後に実装 → リリース待ち...
2. ボタンのクリックも計測したい → また依頼...

GTMを使えば：
- マーケター/アナリストが自分でタグを追加・編集できる
- エンジニアへの依頼が不要（最初の1回だけコードを埋め込んでもらえばOK）
- 間違えてもすぐ元に戻せる（バージョン管理機能あり）

### GTMの3つの構成要素

```
┌─────────────────────────────────────────────────────┐
│                   GTM コンテナ                       │
│                                                      │
│  タグ (Tag)          ：何を送るか                   │
│  　例）GA4イベントタグ → GA4にデータを送信           │
│                                                      │
│  トリガー (Trigger)  ：いつ発火するか                │
│  　例）ページが表示されたとき                         │
│  　　　特定のボタンがクリックされたとき               │
│                                                      │
│  変数 (Variable)     ：何の値を取得するか             │
│  　例）クリックされた要素のテキスト                   │
│  　　　現在のページURL                               │
└─────────────────────────────────────────────────────┘
```

**タグ・トリガー・変数の関係をひとことで：**
> 「**〇〇（トリガー）**のとき、**△△の値（変数）**を使って、**□□（タグ）**を実行する」

実際に手を動かすと理解が深まります。まずこの先のレクチャーを読んで、後の課題で体験してみましょう。

## 3. GA4とGTMの関係

```
ユーザーがボタンをクリック
        ↓
   GTMが検知（トリガー）
        ↓
   GTMがGA4タグを発火
        ↓
   GA4のサーバーにデータ送信
        ↓
   GA4のレポートに表示
```

GTMはあくまで**配送業者**のようなもの。データの中身を決めるのはタグ（GA4など）です。

## 4. プレビューモードとは？

GTMの設定を公開（本番サイトへの反映）する前に動作確認できる機能です。

- 設定したタグが**意図したタイミングで発火しているか**を確認できる
- **GA4のリアルタイムレポートにはデータが届きます**（テスト計測を確認できる）
- 公開前にミスを発見できる

> ⚠️ よくある誤解：「プレビュー中はGA4に何も届かない」は間違いです。
> リアルタイムレポートには反映されます。通常レポートへの集計は数時間かかります。

## 5. UTMパラメータとは？（広告計測の基礎）

GA4でどこから来たユーザーかを正確に把握するために使う**URLにつける目印**です。

```
https://example.com/lp?utm_source=google&utm_medium=cpc&utm_campaign=spring_sale
                       ↑ 流入元        ↑ 媒体           ↑ キャンペーン名
```

| パラメータ | 意味 | 例 |
|-----------|------|-----|
| `utm_source` | 流入元 | google, yahoo, facebook |
| `utm_medium` | 媒体種別 | cpc（有料検索）, organic, email |
| `utm_campaign` | キャンペーン名 | spring_sale, brand_kw |
| `utm_content` | 広告クリエイティブの区別 | banner_a, text_b（同一キャンペーン内で複数の広告を比較するとき） |

広告代理店では、クライアントの広告URLにUTMを付与して「どの広告経由でCVしたか」をGA4で確認します。
UTMが付いていないと、広告経由でのアクセスも「Organic Search」や「Direct（直接流入）」として誤って計上される場合があります（GA4が参照元を自動判定しますが、広告固有のキャンペーン情報は失われます）。

> **命名規則のポイント：** UTMの値はすべて**小文字**で統一しましょう。`Google` と `google` は別々に集計されてしまいます。

---

## 6. 実務の全体像（広告代理店の場合）

```
① 広告にUTMを付与してGA4に流入情報を渡す
② GTMでCVタグを設置（GA4 / Google広告など）
③ GA4でCV数を確認・コンバージョンイベントを登録
④ Google広告にGA4コンバージョンをインポート → 広告の最適化に活用
```

本研修では①〜③をカバーします。④（Google広告との連携）は別途研修で扱います。
