# レクチャー②：環境構築

> 所要時間：約40分（実際に手を動かす）

---

## STEP 1：GA4プロパティを作成する

1. [Google Analytics](https://analytics.google.com/) にアクセス（Googleアカウントでログイン）
2. 左下の「**管理**」をクリック
3. 「**プロパティを作成**」をクリック
4. プロパティ名：任意（例：`STYLE DEMO 研修`）、タイムゾーン：日本、通貨：日本円
5. ビジネス情報は適当に選んでOK
6. 「**ウェブ**」のデータストリームを作成する
   - URL：GitHub PagesのURL（後で決める場合は仮のURLでOK。STEP 4完了後に GA4 → 管理 → データストリーム → ストリームを選択 → ストリームURL を更新してください）
   - ストリーム名：任意
7. 作成後に表示される **「測定ID（G-XXXXXXXXXX）」** をメモしておく

> **測定IDとは？** GA4がどのプロパティにデータを送るかを識別するID。GTMのタグ設定で使います。

---

## STEP 2：GTMコンテナを作成する

1. [Google Tag Manager](https://tagmanager.google.com/) にアクセス
2. 「**アカウントを作成**」をクリック
3. アカウント名：会社名や自分の名前、国：日本
4. コンテナ名：任意（例：`STYLE DEMO`）、ターゲットプラットフォーム：**ウェブ**
5. 作成後にコードスニペットのポップアップが表示されますが、**このサイトにはすでにGTMコードが埋め込まれています**。STEP 3でコンテナIDを書き換えるだけでOKなのでポップアップは閉じてください
6. コンテナID（**GTM-XXXXXXX** 形式）をメモしておく

> **なぜそのまま閉じていいの？** テストサイトのHTMLには `GTM-XXXXXXX` というプレースホルダーが入っています。ここを自分のコンテナIDに書き換えるだけで、コードの貼り付け作業は不要です。

---

## STEP 3：研修用リポジトリをフォークして自分のサイトをデプロイする

研修サイトはGitHubに公開されています。**フォーク**（自分のアカウントにコピー）してGitHub Pagesで公開します。

**3-1. リポジトリをフォークする：**
1. [github.com/tamanuki-nerds/gtm-demo-site](https://github.com/tamanuki-nerds/gtm-demo-site) を開く
2. 右上の「**Fork**」ボタン → 「Create fork」をクリック
3. 自分のアカウントに `gtm-demo-site` リポジトリがコピーされる

**3-2. GitHub Pagesを有効にする：**
1. 自分のリポジトリ（`github.com/あなたのID/gtm-demo-site`）→「Settings」→「Pages」
2. Branch: `main` / Folder: `/ (root)` → 「Save」
3. 数分後に `https://あなたのID.github.io/gtm-demo-site/` でアクセス可能になる

---

## STEP 4：テストサイトにコンテナIDを設定する

フォークしたリポジトリをローカルにクローンし、GTMコンテナIDを書き換えます。

**4-1. クローンする：**
```bash
git clone https://github.com/あなたのID/gtm-demo-site.git
cd gtm-demo-site
```

**4-2. VS CodeでコンテナIDを一括置換する：**
1. VS Codeでクローンしたフォルダを開く
2. `Cmd+Shift+H`（Mac）または `Ctrl+Shift+H`（Windows）で「ファイル内検索と置換」を開く
3. 検索：`GTM-XXXXXXX` → 置換：`GTM-あなたのID`（例：`GTM-AB12CD3`）
4. 「すべて置換」をクリック

**置換後の確認（重要）：**
- 再度 `GTM-XXXXXXX` で検索して0件になっていればOK
- 7ファイル（index / products / product / cart / thanks / contact / form_lp）で置換されているはずです

**4-3. GitHubにプッシュする：**
```bash
git add .
git commit -m "feat: set GTM container ID"
git push
```

> プッシュ後、数分でGitHub Pagesに反映されます。サイトにアクセスして表示を確認してください。

---

## STEP 5：GTMにGA4設定タグを追加する

GTMでGA4計測を始めるには2種類のタグが必要です：

| タグの種類 | 役割 |
|-----------|------|
| **Googleタグ（設定タグ）** | 「どのGA4プロパティに送るか」を定義する親タグ。1つだけ作る。 |
| **GA4イベントタグ** | 「何のイベントを送るか」を定義する子タグ。課題ごとに作る。 |

**設定タグの作成：**
1. GTM管理画面で「**タグ**」→「**新規**」
2. タグの設定をクリック → タグタイプ：**Googleタグ**
3. タグID：STEP 1でメモした `G-XXXXXXXXXX` を入力
4. トリガー：**Initialization - All Pages**（なければ All Pages でもOK）
5. 名前：`GA4 - 設定タグ（Googleタグ）`
6. 「**保存**」

> これが「親タグ」です。この後の課題で作るGA4イベントタグは、すべてこの設定タグに紐付けます。

---

## STEP 6：プレビューモードで接続確認する

1. GTM管理画面で「**プレビュー**」ボタンをクリック
2. テストサイトのURL（`https://...github.io/...`）を入力して「Connect」
3. サイトが別タブで開き、下部に **Tag Assistant** のパネルが表示されればOK
4. トップページを表示して、左のイベント欄に `gtm.js` が表示されていれば接続成功

**パネルが表示されない場合のチェックポイント：**
- STEP 3の置換で `GTM-XXXXXXX` が残っていないか再確認
- GitHub Pagesが反映されていない（もう少し待つ）
- ブラウザの広告ブロッカーを一時的にOFFにする

---

## STEP 7：設定を公開する（バージョン管理）

プレビューで確認した設定は、**公開（Submit）** するまで本番サイトには反映されません。

1. GTM管理画面右上の「**公開**」ボタンをクリック
2. バージョン名：任意（例：`GA4設定タグ追加`）
3. 「**公開**」をクリック

> **バージョン管理とは？** GTMは変更のたびにバージョンが作られます。間違えた場合は「バージョン」タブから以前のバージョンに戻せます。本番サイトでの作業ミスに対する保険です。
> 研修では「プレビューで確認 → 公開」のセットを習慣にしましょう。
