# ハンズオン課題（全7本）

> 所要時間：約2時間
> 前提：[環境構築](./01_setup.md) が完了していること（設定タグ作成・公開まで済んでいること）

各課題の流れ：**GTMで設定 → プレビューで発火確認 → GA4リアルタイムで確認 → 公開**

---

## 課題1：全ページのGA4ページビュー計測

**何をするか：** サイトのどのページにアクセスしても GA4 にページビューが送信されるようにする。

**練習できるGTM機能：** GA4イベントタグ、トリガー（All Pages）、設定タグとイベントタグの関係

### 手順

1. GTMで「タグ → 新規」
2. タグの設定
   - タグタイプ：**Google アナリティクス: GA4 イベント**
   - 設定タグ：環境構築で作った「GA4 - 設定タグ（Googleタグ）」を選択
     - ここで「どのGA4プロパティに送るか」が決まります
   - イベント名：`page_view`
3. トリガー：**All Pages**
4. 名前：`GA4 - ページビュー`
5. 保存 → プレビューで確認

### 発火確認マトリクス

| ページ | このタグの発火 |
|--------|---------------|
| index.html | ✅ 発火する |
| products.html | ✅ 発火する |
| thanks.html | ✅ 発火する |
| contact.html | ✅ 発火する |

### GA4リアルタイムレポートでの確認

1. GA4管理画面 → レポート → リアルタイム
2. 「イベント別ユーザー数」に `page_view` が表示されればOK

> ⚠️ **二重計測に注意：** GA4のデータストリーム設定で「拡張計測機能」の `page_view` が有効になっている場合、GA4が自動でpage_viewを計測します。GTMでも手動でpage_viewタグを作ると**2つのpage_viewが送信**されてしまいます。研修環境では問題ありませんが、実務で設定する際は拡張計測機能の状態を確認してどちらか1つに統一してください。

## 課題2：購入完了ページだけCV計測タグを発火させる

**何をするか：** `thanks.html` にアクセスしたときだけCV計測タグが発火するようにする。「絞り込む」のが重要。

**練習できるGTM機能：** トリガー（ページビュー・一部のページ）、変数（Page URL）

### 手順

1. 組み込み変数を有効にする
   - GTM → 変数 → 組み込み変数の「設定」→「ページ」カテゴリの `Page URL` にチェック
2. GTMで「タグ → 新規」
3. タグの設定
   - タグタイプ：Google アナリティクス: GA4 イベント
   - 設定タグ：「GA4 - 設定タグ」
   - イベント名：`purchase`
4. トリガー → 「新規」
   - トリガーの種類：**ページビュー**
   - 「一部のページビュー」を選択
   - 条件：`Page URL` ／ `含む` ／ `thanks.html`
   - 名前：`トリガー - 購入完了ページ`
5. 名前：`GA4 - 購入CV`
6. 保存

### 発火確認マトリクス

| ページ | このタグの発火 |
|--------|---------------|
| index.html | ❌ 発火しない |
| products.html | ❌ 発火しない |
| cart.html | ❌ 発火しない |
| **thanks.html** | ✅ **発火する** |

> **確認方法：** プレビューで直接 `thanks.html` のURLを入力して接続し、ページを開いたときに発火するか確認しましょう。

### GA4での追加作業

- GA4でこのイベントをコンバージョンに登録する
  - GA4 → 管理 → イベント → `purchase` の右にある「コンバージョンとしてマーク」をON
- ⚠️ **注意：** コンバージョン登録前のデータは遡及されません。登録後から集計が始まります。

## 課題3：「カートに追加」クリック計測 + dataLayer変数

**何をするか：** 商品詳細ページの「カートに追加」ボタンのクリックを計測し、商品名・価格もGA4に渡す。

**練習できるGTM機能：** クリックトリガー、クリックID変数、カスタムイベントトリガー、**dataLayer変数（重要）**

### 「組み込み変数」と「dataLayer変数」の違い

GTMには2種類の変数があります。この課題で両方を体験します。

| 種類 | 取得できるもの | 設定の手間 |
|------|-------------|-----------|
| **組み込み変数** | GTMが自動で取得できる値（Click ID・Click Text・Page URLなど） | 変数一覧でチェックONするだけ |
| **dataLayer変数** | サイトのJS側が `dataLayer.push()` で明示的に送った値（商品名・価格など） | GTMで変数を作成し、JS側のキー名と一致させる必要あり |

> **STEP Aとが同時に発火する理由：**「カートに追加」ボタンをクリックすると2つのことが起きます。①GTMがクリックを検知（Click ID = `btnAddToCart` → STEP Aのトリガー発火）、②同時にページのJSが `dataLayer.push({event: "add_to_cart", ...})` を実行（→ STEP Bのトリガー発火）。1回のクリックで2つのイベントがGA4に送信されます。

### STEP A：クリックトリガーでボタン計測

1. 組み込み変数を有効にする
   - 「クリック」カテゴリの `Click ID`、`Click Text`、`Click Classes` にチェック
2. トリガーを作成
   - トリガーの種類：**クリック - すべての要素**
   - 「一部のクリック」を選択
   - 条件：`Click ID` ／ `等しい` ／ `btnAddToCart`
   - 名前：`トリガー - カートに追加クリック`
3. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`add_to_cart_click`
   - イベントパラメータ：`button_text` = `{{Click Text}}`
   - 名前：`GA4 - カートに追加クリック`

### STEP B：dataLayer変数で商品情報を取得する

「カートに追加」ボタンを押すと、サイトのJSが自動的に以下のデータをdataLayerに送っています：

```javascript
dataLayer.push({
  event: "add_to_cart",
  product_name: "オーバーサイズTシャツ",
  product_price: 4400,
  selected_size: "M"
})
```

このデータをGTMで受け取るには「dataLayer変数」を作る必要があります。

1. GTM → 変数 → ユーザー定義変数 → 「新規」
   - 変数タイプ：**データレイヤーの変数**
   - データレイヤーの変数名：`product_name`
   - 名前：`dlv - product_name`
2. 同様に `product_price`（名前：`dlv - product_price`）も作成

3. カスタムイベントトリガーを作成
   - トリガーの種類：**カスタムイベント**
   - イベント名：`add_to_cart`（dataLayerのevent値）
   - 名前：`トリガー - add_to_cart イベント`

4. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`add_to_cart`
   - イベントパラメータ：
     - `product_name` = `{{dlv - product_name}}`
     - `product_price` = `{{dlv - product_price}}`
   - トリガー：「トリガー - add_to_cart イベント」
   - 名前：`GA4 - カートに追加（dataLayer）`

### 発火確認マトリクス

| 操作 | タグの発火 |
|------|-----------|
| 商品詳細でボタンをクリック（サイズ選択前） | ❌ アラートが出て発火しない |
| 商品詳細でサイズ選択 → ボタンをクリック | ✅ 両方のタグが発火 |
| 商品一覧ページ上のクリック | ❌ `btnAddToCart` IDがないので発火しない |

### プレビューでの確認ポイント

- 左のイベント欄に「Click」と「カスタムイベント（add_to_cart）」の2つが表示される
- カスタムイベントのVariable欄で `dlv - product_name` の値に商品名が入っているか確認

#### GA4 DebugViewでパラメータを確認する

GA4管理画面 → **管理 → DebugView** を開くと、GTMプレビュー中に送信されたイベントのパラメータを1件ずつリアルタイムで確認できます。`add_to_cart` イベントをクリックして `product_name` の値が届いているか確認しましょう。

> **カスタムディメンションの登録：** `product_name` などのイベントパラメータはGA4に届いていても、**カスタムディメンションとして登録しないとレポート画面には表示されません**。登録方法：GA4 → 管理 → カスタム定義 → カスタムディメンション → 「カスタムディメンションを作成」→ 範囲：イベント / イベントパラメータ：`product_name`。DebugViewや探索レポートでは登録不要で値を確認できます。

## 課題4：外部リンクの離脱クリック計測

**何をするか：** サイト内のリンクではなく、外部サイトへのリンククリックだけを計測する。

**練習できるGTM機能：** トリガー（クリック - リンクのみ）、変数（Click URL）

### 手順

1. 組み込み変数で `Click URL` を有効にする
2. トリガーを作成
   - トリガーの種類：**クリック - リンクのみ**
   - 「一部のリンククリック」を選択
   - タグの配信を待つ：チェックON、待機時間：2000ms
   - 条件：`Click URL` ／ `含まない` ／ `github.io`
   - 名前：`トリガー - 外部リンク離脱`
3. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`outbound_click`
   - イベントパラメータ：`link_url` = `{{Click URL}}`
   - 名前：`GA4 - 外部リンク離脱`

### 発火確認マトリクス

| クリック対象 | 発火 |
|------------|------|
| トップページの「アナグラム：GTM入門」リンク | ✅ 発火 |
| トップページの「Google公式：GTMヘルプ」リンク | ✅ 発火 |
| ナビの「商品一覧」リンク（内部リンク） | ❌ 発火しない |

> **練習場所：** トップページ下部の「参考リンク」セクション

## 課題5：スクロール50%でイベントを送信する

**何をするか：** 商品詳細ページを半分以上スクロールしたユーザーにイベントを送信する。

**練習できるGTM機能：** トリガー（スクロール距離）、組み込み変数（Scroll Depth Threshold）

### 手順

1. 組み込み変数で `Scroll Depth Threshold` を有効にする（スクロールカテゴリ）
2. トリガーを作成
   - トリガーの種類：**スクロール距離**
   - 縦方向スクロール：チェックON、割合：`50`（50%）
   - このトリガーの発生場所：「一部のページ」
   - 条件：`Page URL` ／ `含む` ／ `product.html`
   - 名前：`トリガー - スクロール50% 商品詳細`
3. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`scroll`
   - イベントパラメータ：`percent_scrolled` = `{{Scroll Depth Threshold}}`
     - 固定値50ではなく変数を使うと、将来75%・90%のトリガーを追加しても同じタグを使い回せます
   - 名前：`GA4 - スクロール50%`

### 発火確認マトリクス

| ページ・操作 | 発火 |
|------------|------|
| 商品詳細ページを50%以上スクロール | ✅ 1回だけ発火 |
| 商品詳細ページを100%スクロール後に戻る | ❌ 再発火しない（1回のみ） |
| トップページをスクロール | ❌ 発火しない |

## 課題6：フォーム送信のCV計測 + タイマー（30秒滞在）

### 課題6-A：フォーム送信CV

**何をするか：** お問い合わせフォームが送信されたときにCV計測タグを発火させる。

**練習できるGTM機能：** トリガー（フォーム送信）、変数（Form ID）

#### 手順

1. 組み込み変数で `Form ID` を有効にする（フォームカテゴリ）
2. トリガーを作成
   - トリガーの種類：**フォーム送信**
   - 「一部のフォーム」を選択
   - タグの配信を待つ：チェックON
   - 条件：`Form ID` ／ `等しい` ／ `contactForm`
   - 名前：`トリガー - お問い合わせフォーム送信`
3. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`generate_lead`
   - 名前：`GA4 - お問い合わせCV`

#### 発火確認マトリクス

| 操作 | 発火 |
|------|------|
| 全項目入力 → 送信ボタンクリック | ✅ 発火する |
| 空欄で送信ボタンクリック（バリデーションエラー） | ❌ 発火しない |

> **補足：** サイトのJSが `e.preventDefault()` でデフォルトの送信を止めているため、実際にはページ遷移しません。GTMのフォーム送信トリガーは送信イベント自体を検知するので、遷移なしでも発火します。

---

### 課題6-B：30秒滞在タイマー

**何をするか：** お問い合わせページに30秒以上滞在したユーザーにイベントを送信する。

**練習できるGTM機能：** トリガー（タイマー）

> **実務での使い場面：** 30秒以上滞在したユーザーは「真剣にページを読んでいる」と判断できます。このイベントをGA4でオーディエンスとして保存し、リターゲティング広告で「ほぼ興味あり層」に対して積極配信するという使い方が代理店では頻出です。
>
> ⏭️ **確認が面倒な場合は先に進んでOK。** 30秒待つ必要があるため、課題7を先に進め、時間が余ったら戻って確認しましょう。

#### 手順

1. トリガーを作成
   - トリガーの種類：**タイマー**
   - 間隔：`30000`（30秒 = 30,000ミリ秒）
   - 上限：`1`（1回だけ発火）
   - このトリガーの発生場所：「一部のページ」
   - 条件：`Page URL` ／ `含む` ／ `contact.html`
   - 名前：`トリガー - 30秒滞在（お問い合わせ）`
2. タグを作成
   - タグタイプ：GA4 イベント / イベント名：`long_engagement`
   - イベントパラメータ：`time_seconds` = `30`
   - 名前：`GA4 - 30秒滞在`

#### 確認方法

- お問い合わせページを開いて30秒待つ
- プレビューの左イベント欄に「gtm.timer」が表示 → タグが発火すればOK

## 課題7：LP型フォームのCV計測（dataLayerカスタムイベント応用）

**何をするか：** URLが変わらないランディングページ（`form_lp.html`）のフォーム送信をCV計測する。
課題2とは違い「URLが変わらない」ため、ページビュートリガーではなくカスタムイベントトリガーで計測します。

**サイト：** `form_lp.html`（架空のBtoBサービス「TimeSync」の資料請求LP）
**練習できるGTM機能：** カスタムイベントトリガー、dataLayer変数、GA4イベントパラメータ

### このページで発生するdataLayerイベント

| イベント名 | タイミング | 渡すデータ |
|-----------|-----------|-----------|
| `form_start` | フォームへの最初のフォーカス | `form_id` |
| `form_validation_error` | 空欄で送信しようとしたとき | `form_id` |
| `lp_cv` | フォーム送信成功 | `form_id`, `company_size`, `has_phone`, `currency`, `value` |

> **今回タグを設定するのは `lp_cv` だけでOKです。** `form_start` と `form_validation_error` はサイト側が自動で送出しますが、今回のGTM設定では受け取るタグを作りません。発展課題として時間が余ったら挑戦してみましょう。
>
> **`has_phone` について：** 電話番号を入力したかどうかの真偽値（`true` / `false`）が送られます。「電話番号を教えてくれた＝温度感が高い」とみなし、営業優先度の区別に使えます。今回のタグ設定では不要ですが、実務での活用イメージとして参考にしてください。

### 手順

1. dataLayer変数を作成
   - 変数タイプ：データレイヤーの変数
   - `company_size`（名前：`dlv - company_size`）

2. カスタムイベントトリガーを作成
   - トリガーの種類：**カスタムイベント**
   - イベント名：`lp_cv`
   - 名前：`トリガー - LP資料請求CV`

3. タグを作成（`lp_cv` だけ計測すればOK）
   - タグタイプ：GA4 イベント / イベント名：`generate_lead`
   - イベントパラメータ：
     - `lead_source` = `timesync_lp`（固定値）
     - `company_size` = `{{dlv - company_size}}`
   - トリガー：「トリガー - LP資料請求CV」
   - 名前：`GA4 - LP資料請求CV`

### 発火確認マトリクス

| 操作 | 発火 |
|------|------|
| LPを開いただけ | ❌ 発火しない（URLが変わっても変わらなくても） |
| フォームへの入力開始 | `form_start` イベントが発火（今回はタグ設定不要） |
| 空欄で送信 → バリデーションエラー | ❌ `lp_cv` は発火しない |
| 全項目入力 → 送信 | ✅ `lp_cv` が発火 → タグが発火 |

### このページで学べる重要概念

```
課題2（thanks.html）：  URL変化をトリガーにしたCV計測
課題7（form_lp.html）： dataLayerカスタムイベントをトリガーにしたCV計測
                        ↑ SPAやLP型で最も実務頻出のパターン
```

### GA4での確認

- リアルタイムで `generate_lead` イベントを確認
- **GA4 DebugView**（管理 → DebugView）で `company_size` の値が届いているか確認（例：`11-50`）
- コンバージョンに登録してみよう

> **カスタムディメンション：** `company_size` をGA4のレポートに表示したい場合は、課題3と同様にカスタムディメンションを登録してください（GA4 → 管理 → カスタム定義 → カスタムディメンション → イベントパラメータ：`company_size`）。

> **発展課題（時間が余った方へ）：** `form_start` と `form_validation_error` イベントもGTMで受け取るタグを作ってみましょう。「フォームを触ったが送信しなかった離脱率」や「バリデーションエラー頻度」を計測する、実務でよく求められる設定です。
