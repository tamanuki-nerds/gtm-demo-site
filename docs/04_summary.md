## 自己チェックリスト

以下の項目を自分の言葉で説明できるか確認してください。

### 基礎知識

- [ ] GA4の「イベント」とは何か、一言で説明できる
- [ ] GTMの「タグ・トリガー・変数」の役割を、それぞれ一言で説明できる
- [ ] GTMを使う理由（エンジニアに頼まなくて済む）を説明できる
- [ ] GTMのプレビューモードで何を確認するか説明できる
- [ ] UTMパラメータが何のためにあるか説明できる

### GTM操作

- [ ] GA4設定タグ（Googleタグ）とGA4イベントタグの違いが分かる
- [ ] All Pagesトリガーと、URL条件付きページビュートリガーの違いが分かる
- [ ] クリックトリガーで特定のボタンだけに絞り込む方法が分かる
- [ ] dataLayer変数の作り方・使い方が分かる
- [ ] プレビューで確認した後に「公開（Submit）」が必要なことが分かる

### GA4計測

- [ ] リアルタイムレポートでイベントが届いていることを確認できる
- [ ] DebugViewでイベントパラメータの値を確認できる
- [ ] コンバージョンイベントの登録方法が分かる
- [ ] コンバージョン登録前のデータは遡及されないことを知っている
- [ ] URLが変わらないLP型でカスタムイベントトリガーを使う理由を説明できる

## よくある失敗パターン

実務で新人がやりがちなミスを事前に把握しておきましょう。

| ミス | 対処法 |
|------|--------|
| プレビューで確認したのに本番に反映されない | GTMの「公開（Submit）」を忘れている |
| 全クリックが計測されてしまう | トリガーの絞り込み条件が不足している |
| GA4にデータが届いているのにレポートに出ない | 通常レポートは数時間遅れる。リアルタイムで確認する |
| コンバージョンを登録したが過去のデータが0 | 登録前のデータは遡及されない。これは仕様 |
| dataLayer変数が空（undefined）になる | GTMのdataLayer変数名がapp.jsのキー名と一致しているか確認 |
| フォーム送信トリガーが発火しない | `Form ID` の組み込み変数が有効になっているか確認 |
| LP型フォームでフォーム送信トリガーを使ってしまう | URLが変わらないLP・SPA型はフォーム送信トリガーではなく**カスタムイベントトリガー**を使う。`e.preventDefault()` でHTMLのsubmitイベントが止まる場合、GTMのフォームトリガーが検知できないことがある |
| UTMの値が大文字混じりでレポートが汚染される | `Google` と `google` は別流入として集計される。UTMの値はすべて**小文字**で統一するルールをチームで決める |

## 実務に向けたネクストステップ

1. **Google広告コンバージョンの設定**
   GA4のコンバージョンをGoogle広告にインポートして、広告の入札最適化に使えるようにする。広告代理店では必須スキル。

2. **GTMを使ったエンジニアへの設置依頼**
   実案件では自分でHTMLに貼れないケースがほとんど。GTMスニペット（2つのコード）をエンジニアに渡す際の注意点（`<head>`最上部に1つ目、`<body>`直後に2つ目）を確認しておく。

3. **UTMの実運用**
   広告URLへのUTM付与ルールをチームで統一する。`utm_source`, `utm_medium`, `utm_campaign` の命名規則が揃っていないとGA4のレポートが汚染される。
